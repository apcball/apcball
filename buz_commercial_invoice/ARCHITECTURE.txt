SEQUENCE GENERATION LOGIC
==========================

Thread-Safe Atomic Counter:

    User Action: Check "Generate Commercial Invoice"
              ↓
    Call: sale_order.write({'commercial_invoice_enabled': True})
              ↓
    Method: write(vals) checks 'commercial_invoice_enabled'
              ↓
    Condition: if enabled AND no number yet
              ↓
    Call: _get_commercial_invoice_number()
              ↓
    Calls: ir.sequence.next_by_code('commercial.invoice.sequence')
              ↓
    Database: 
    ┌─ BEGIN TRANSACTION
    ├─ Lock ir.sequence record
    ├─ Read: number_next = 1
    ├─ Generate: "CIV-" + "000001" = "CIV-000001"
    ├─ Update: number_next = 2
    └─ COMMIT TRANSACTION
              ↓
    Result: 'CIV-000001' returned
              ↓
    Store: record.commercial_invoice_number = 'CIV-000001'
              ↓
    Save to DB and display in form


ERROR HANDLING FLOW
===================

User Action: Click "Print Commercial Invoice"
             ↓
    Validation 1: commercial_invoice_enabled == True?
             ├─ No → Raise: "Commercial Invoice is not enabled..."
             └─ Yes → Continue
             ↓
    Validation 2: commercial_invoice_number exists?
             ├─ No → Raise: "CIV number was not generated..."
             └─ Yes → Continue
             ↓
    Generate: Report PDF
             ├─ Success → Display PDF to user
             └─ Error → Show error message

KEY DESIGN DECISIONS
====================

1. ✓ Sequence-based numbering
   - Thread-safe atomic operations
   - No conflicts in multi-user environments
   - Easy to reset if needed

2. ✓ Checkbox-based activation
   - Simple user interface
   - Clear intent
   - Easy to understand

3. ✓ Read-only CIV number
   - Prevents manual manipulation
   - Maintains data integrity
   - Auditable

4. ✓ Computed amount_text
   - Auto-updates with amounts
   - Always accurate
   - Reduces manual errors

5. ✓ Backward compatibility
   - Existing invoice workflows unchanged
   - Legacy support maintained
   - Gradual migration possible

6. ✓ Independent from invoicing
   - Works without posting invoice
   - Suitable for advance notifications
   - Flexible use cases
