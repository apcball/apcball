All In One Backdate Advanced

Comprehensive backdating solution for Odoo 17

🚀 Key Features

This module provides comprehensive backdating functionality for various Odoo documents with proper security controls and audit trails.

📄 Document Support

  • Customer Invoices
  • Vendor Bills
  • Customer Payments
  • Vendor Payments
  • Journal Entries
  • Bank Statements
  • Sale Orders
  • Purchase Orders
  • Stock Pickings

🔐 Security & Permissions

  • User-based permissions
  • Manager override capabilities
  • Date range restrictions
  • Fiscal year lock validation
  • Document type controls

📊 Audit & Compliance

  • Complete audit trail
  • Reason tracking
  • User activity logs
  • Date change history
  • Compliance reporting

⚙️ Configuration

  • Configurable date limits
  • Enable/disable by document type
  • Mandatory reason settings
  • Company-specific rules
  • Automated log cleanup

⚠️ Important Notes

  • Backdating should be used with caution as it affects financial reports
  • All backdate operations are logged for audit purposes
  • Proper user permissions must be configured before use
  • Consider fiscal year locks and accounting periods

🛠️ Installation & Setup

  1. Install the module from Apps menu
  2. Go to Settings > General Settings > Backdate section
  3. Configure your backdate preferences
  4. Assign appropriate user groups in Settings > Users & Companies > Groups
  5. Start using backdate functionality on supported documents

👥 User Groups

  • Backdate User: Can backdate documents with restrictions
  • Backdate Manager: Can backdate any document without restrictions

📈 Usage

Once configured, users with appropriate permissions will see a "Backdate" button on supported documents. The system will:

  • Validate the new date against configured restrictions
  • Update the document date and related records
  • Log the operation for audit purposes
  • Maintain data integrity throughout the process
